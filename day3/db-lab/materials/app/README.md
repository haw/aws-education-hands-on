# 社員管理システム (Employee Management System) - Cosmos Edition

このプロジェクトは、AWS ハンズオン（EC2 と RDS の連携）における学習用の教材として作成された、Node.js + Express 製の**社員管理システム**です。
完全な CRUD（登録・一覧・編集・削除）機能を備えており、バックエンドを汚さずにフロントエンドのみで「コスモ（小宇宙）」を感じさせるド派手な宇宙空間テーマの UI を構築しています。

---

## 📋 プロジェクトの全体像と設計思想（次期担当開発者/AI向け）

次に本プロジェクトのメンテナンスや改修を担当する AI および開発者は、以下の設計思想と背景を理解してください。

### 1. 教材としてのバックエンドの維持
- **`server.js` は極めてシンプルかつピュアな状態を維持する**必要があります。
- 受講者が混乱しないよう、ポートは本来の `3000` 固定とし、複雑なエラーハンドリングミドルウェアや非同期キャッチなどの追加コードは一切記述していません。
- データベース接続情報は、AWS 上では Secrets Manager (`Mydbsecret`) から自動取得し、ローカル環境や取得失敗時は環境変数またはデフォルト値にフォールバックする設計（`config.js`）になっています。

### 2. UI/UX 側の「コスモ（小宇宙）」演出の分離
- すべてのド派手な演出（星空背景、流れ星、送信時のビッグバンチャージエフェクト）は、**`views/index.ejs`（JavaScript）と `public/style.css` のみで完結**しています。
- **送信エフェクトの仕組み**:
  登録・更新・削除ボタンをクリックした際、`index.ejs` の JavaScript がフォーム送信イベントを一時的（1.5秒間）に `preventDefault()` で保留します。その間にフルスクリーンの「ワープ・星雲・コスモチャージ」エフェクトを走りらせ、クライマックスの「ビッグバン（白い閃光）」の瞬間に実際の `form.submit()` を呼び出すことで、バックエンド（`server.js`）のルーティングを変更することなくリッチな演出を実現しています。

### 3. トラブルシューティングの歴史（ポート 3000 競合問題）
- ローカル環境で `localhost:3000` にアクセスした際、ブラウザに `Error: ENOENT: ... .org.chromium.Chromium...` というエラーが表示される場合があります。
- これは Express サーバーのエラーではなく、**ポート 3000 を自動フォワードしようとしている外部ツール（VS Code の自動ポート転送や、バックグラウンドで動いている Tauri アプリ等のプロキシ）が内部でクラッシュしていること**が原因です。
- この現象が発生した場合は、プロキシとなっている該当プロセス（例: `SwiftQuot` など）を `lsof -nP -iTCP:3000` で特定して `kill` するか、一時的に `server.js` のポートを `3005` などに変更してプロキシをバイパスしてください。

## 🛠️ 技術スタック
- **Runtime**: Node.js (`v18` 以上推奨)
- **Web Framework**: Express 5 (`^5.2.1`)
- **Template Engine**: EJS (`^6.0.1`)
- **Body Parser**: Body-parser (`^2.2.2`)
- **Database Driver**: MySQL2 (`^3.6.0`)
- **AWS Integration**: `@aws-sdk/client-secrets-manager` (`^3.700.0`)

## 🚀 ローカル環境での動作確認手順

ローカル PC 環境で動作確認を行いたい場合は、以下の手順に従って環境を構築・起動してください。

### 前提条件
- **Node.js** がインストールされていること (`node --version` で確認)
- **Docker** が起動していること (`docker --version` で確認)

---

### ステップ 1: ローカル検証用 MySQL 8.4 の起動 (Docker)
Docker を使用して、検証用の MySQL 8.4.10 コンテナをバックグラウンドで起動します。
```bash
docker run --name db-lab-mysql \
  -e MYSQL_ROOT_PASSWORD=password123 \
  -e MYSQL_DATABASE=employeedb \
  -e MYSQL_USER=admin \
  -e MYSQL_PASSWORD=password123 \
  -p 3306:3306 \
  -d mysql:8.4
```
*(※すでに同名のコンテナが存在するというエラーが出た場合は、先に後述する「後片付け」コマンドで古いコンテナを削除してください)*

### ステップ 2: 依存パッケージのインストール
プロジェクトディレクトリで以下を実行し、必要な npm パッケージをインストールします。
```bash
npm install
```

### ステップ 3: データベースとテーブルの初期化
環境変数を指定して初期化スクリプトを実行し、データベースの作成、テーブル定義、およびサンプルの初期データを投入します。
```bash
APP_DB_HOST=127.0.0.1 \
APP_DB_USER=admin \
APP_DB_PASSWORD=password123 \
APP_DB_NAME=employeedb \
node init_db.js
```
成功すると、ターミナルに `🎉 データベース初期化完了` と表示されます。

### ステップ 4: アプリケーションサーバーの起動
環境変数を指定して、Express サーバーを起動します。
```bash
APP_DB_HOST=127.0.0.1 \
APP_DB_USER=admin \
APP_DB_PASSWORD=password123 \
APP_DB_NAME=employeedb \
npm start
```
成功すると、`サーバーがポート3000で起動しました` および `データベースに接続しました` と出力されます。

### ステップ 5: ブラウザでの確認
ブラウザを起動し、アドレスバーに直接以下のいずれかの URL を入力してアクセスしてください。
- **[http://localhost:3000](http://localhost:3000)**
- **[http://127.0.0.1:3000](http://127.0.0.1:3000)**

画面を開いて数秒待つと、背景に美しい**流れ星**が優雅にすーっと流れます。新規登録、編集、削除を行うと、**「コスモの覚醒＆ビッグバン（削除時はブラックホール消滅）」**のエフェクトが走ります。

---

## 🧹 後片付け（確認終了後）

検証が終わりましたら、以下の手順でプロセスとコンテナをクリーンアップしてください。

1. アプリケーションを起動しているターミナルで **`Ctrl + C`** を押し、Node.js プロセスを終了します。
2. 起動した Docker コンテナを停止し、削除します。
   ```bash
   docker stop db-lab-mysql
   docker rm db-lab-mysql
   ```
